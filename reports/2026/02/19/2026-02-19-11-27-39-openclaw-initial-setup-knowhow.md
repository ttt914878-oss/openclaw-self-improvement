# OpenClawの初期設定Tips・運用ノウハウ（2026時点）

## 3行要約
- OpenClawは、**役割定義（SOUL/IDENTITY）を最初に明確化**し、ツール権限を最小化すると安定運用しやすい。  
- 日々の運用では、**HEARTBEATでの定期点検・コンテキスト整理・週次レビュー**を回すと品質劣化を防げる。  
- 典型障害（ドリフト、ツール詰まり、思考ループ）は、**再初期化手順・タイムアウト/リトライ・上限ルール**を先に決めておくと復旧が速い。  

## 実践チェックリスト

### 1) 初期設定
- [ ] `SOUL.md` に「やること / やらないこと（禁止事項）」を明記する
- [ ] `IDENTITY.md` で担当領域を絞る（万能化しすぎない）
- [ ] `TOOLS.md` で危険操作を制限し、最小権限で開始する
- [ ] `BOOTSTRAP.md` で依存関係・環境変数・初期動作確認を自動化する
- [ ] ログと成果物の保存先を固定し、命名規則（日時付き）を統一する

### 2) 運用
- [ ] `HEARTBEAT.md` を使い、定期チェック項目（通知・予定・失敗ジョブ）を定義する
- [ ] `USER.md` を都度更新し、優先度や好みの変化を反映する
- [ ] 不要ログ/中間ファイルを定期アーカイブし、コンテキスト肥大化を防ぐ
- [ ] 週次で出力サンプルをレビューし、プロンプト/ルールを微調整する
- [ ] 失敗ケース（原因・対処・再発防止）をナレッジ化して再利用する

### 3) トラブル対策
- [ ] 役割逸脱（ドリフト）時の再読込手順（SOUL/IDENTITY再適用）を定義する
- [ ] ツール停止に備え、タイムアウト値と段階的リトライ（例: 20s→40s）を設定する
- [ ] 429や一時障害時の待機戦略（指数バックオフ・最大回数）を定義する
- [ ] 思考ループ対策としてステップ上限と「人間へエスカレーション」条件を設ける
- [ ] 重要操作（削除/外部送信）は確認ゲートを必須化する
